# Saju Love Match 💘

생년월일과 태어난 시각을 입력하면, 등록된 아이돌·배우·가수와의 **유사연애형 사주 궁합**을 계산해 순위로 보여주는 오락용 웹앱입니다.

- 프레임워크 없음
- 서버 없음
- `index.html` + `idols.json`만으로 작동
- 모든 계산은 브라우저 안에서 실행
- 입력한 생년월일과 시각은 외부 서버로 전송되지 않음

> 이 프로젝트는 전통 명리학 요소를 연애·유사연애 콘셉트에 맞게 재구성한 오락용 모델입니다.  
> 과학적 예측이나 실제 관계의 보증하지 않습니다.

# 1. 입력값

## 생년월일

`YYYYMMDD` 형식의 8자리 숫자를 입력합니다.

```text
19971231
```

입력값을 이용해 다음을 계산합니다.

- 년주
- 월주
- 일주
- 일간
- 일지
- 오행 분포
- 신강·신약 상태

---

## 태어난 시각

24시간 형식으로 선택 입력합니다.

```text
12:31
```

태어난 시각을 입력하면:

- 사용자의 시주
- 사용자의 시지
- 사용자 시지와 상대 일지의 관계

가 점수에 반영됩니다.

상대 인물의 태어난 시각은 대부분 공개되지 않았기 때문에, 상대는 생년월일 기준 년·월·일주만 사용합니다.

태어난 시각을 입력하지 않으면 **시주 궁합 점수 60점**을 중립값으로 사용합니다.

---

## 사용자 성별

성별은 선택 사항입니다.

| 선택 | 적용 방식 |
|---|---|
| 여성 | 관성(정관·편관)을 이성 인연 십신으로 소폭 가중 |
| 남성 | 재성(정재·편재)을 이성 인연 십신으로 소폭 가중 |
| 선택 안 함 | 성별 중립 점수 사용 |

성별 선택은 순위 전체를 강제로 바꾸는 요소가 아니라, 전통 명리의 이성 인연 개념을 **조금 더 반영하는 보정값**입니다.

### 여성 선택 시

| 십신 | 배수 |
|---|---:|
| 정관 | ×1.25 |
| 편관 | ×1.10 |
| 정재·편재 | ×0.95 |
| 나머지 | ×1.00 |

### 남성 선택 시

| 십신 | 배수 |
|---|---:|
| 정재 | ×1.25 |
| 편재 | ×1.10 |
| 정관·편관 | ×0.95 |
| 나머지 | ×1.00 |

# 2. 궁합 계산 항목

최종 기본점수는 6개 항목을 각각 0~100점으로 계산한 뒤 비중에 따라 합산합니다.

```text
기본 종합점수
= 연애 십신 × 35%
+ 일지 궁합 × 20%
+ 천간합 × 15%
+ 오행 밸런스 × 15%
+ 신강·신약 보완 × 10%
+ 시주 궁합 × 5%
```

| 항목 | 비중 |
|---|---:|
| 💕 연애 십신 | 35% |

+ 운명 보너스

# 4. 설렘·안정 점수

최종 순위는 LOVE MATCH 종합점수 기준으로 정렬하지만, 결과 카드에는 설렘과 안정 점수도 함께 표시합니다.

## 설렘 점수

```text
설렘
= 연애 십신 × 40%
+ 천간합 × 30%
+ 일지 궁합 × 20%
+ 시주 궁합 × 10%
```

추가 보정:

- 편재·편관·상관이면 설렘 가점
- 일지 충이면 강한 긴장감 보정

## 안정 점수

```text
안정
= 일지 궁합 × 35%
+ 연애 십신 × 25%
+ 오행 밸런스 × 20%
+ 신강·신약 보완 × 15%
+ 시주 궁합 × 5%
```

예시:

```text
설렘 96.30
안정 72.40
종합 88.75
```

같은 종합점수라도 불꽃형인지 안정형인지 확인할 수 있습니다.

# 7. 카드 등급

최종 LOVE MATCH 점수에 따라 카드 디자인이 달라집니다.

| 점수 | 카드 |
|---|---|
| 90 미만 | 기본 카드 |
| 90~99.99 | 💗 HIGH MATCH |
| 100~105.99 | 💜 DESTINED MATCH |
| 106 이상 | ✦ SOULMATE |

- `HIGH MATCH`: 핑크 계열 강조
- `DESTINED MATCH`: 보라 그라데이션과 광택 효과
- `SOULMATE`: 골드 계열과 반짝임 효과

기기에서 동작 줄이기 설정을 사용하면 애니메이션이 자동으로 줄어듭니다.

# 8. 인물 데이터 추가

인물 데이터는 프로젝트 루트의 `idols.json`에서 관리합니다.

## 데이터 형식

```json
{"name": "강유찬",  "group": "A.C.E",  "agency": "기타",  "gender": "M",  "dob": "1997-12-31",  "cat": "K-idol"}
```

## 필드 설명

| 필드 | 설명 | 예시 |
|---|---|---|
| `name` | 화면에 표시할 활동명 | `강유찬` |
| `group` | 그룹 또는 활동 구분 | `A.C.E` |
| `agency` | 소속사 또는 분류 | `기타` |
| `gender` | 상대 인물 성별 | `M` 또는 `F` |
| `dob` | 양력 생년월일 | `1997-12-31` |
| `cat` | 카테고리 | `K-idol` |

## 여러 명 추가 예시

```json
[
  { "name": "강유찬",    "group": "A.C.E",    "agency": "기타",    "gender": "M",    "dob": "1997-12-31",    "cat": "K-idol"  },
  { "name": "스즈키 노부유키",    "group": "스즈키 노부유키",    "agency": "기타",    "gender": "M",    "dob": "1992-10-14",    "cat": "J-idol"  }
]
```

## JSON 작성 시 주의

- 파일 전체는 `[`와 `]`로 감쌉니다.
- 각 인물은 `{`와 `}`로 감쌉니다.
- 마지막 인물을 제외하고 각 객체 뒤에는 쉼표가 필요합니다.
- 모든 키와 문자열은 큰따옴표를 사용합니다.
- 날짜는 반드시 `YYYY-MM-DD` 형식으로 입력합니다.
- 남성은 `M`, 여성은 `F`로 입력합니다.
- 동일한 `name + group` 조합은 중복으로 취급하는 것이 좋습니다.

# 9. 데이터 검사

프로젝트에 `tools/validate.js`가 있다면 Node.js로 데이터 검사를 할 수 있습니다.

프로젝트 루트에서 실행:

```bash
node tools/validate.js
```

검사 항목:

- JSON 문법이 올바른지
- 최상위 구조가 배열인지
- 필수 필드가 모두 존재하는지
- 각 필드가 문자열인지
- 이름과 그룹이 비어 있지 않은지
- 성별이 `M` 또는 `F`인지
- 생년월일이 실제 존재하는 날짜인지
- 같은 `name + group`이 중복되었는지

성공 예시:

```text
Validation passed: 555 records
```

오류 예시:

```text
Validation failed: record 12 has an invalid dob: 1997-13-31
```

아이폰만 사용하는 경우 GitHub Codespaces 또는 컴퓨터의 Node.js 환경에서 검사할 수 있습니다.

# 10. 로컬 실행

`index.html`은 `idols.json`을 `fetch()`로 읽기 때문에, 일부 브라우저에서는 파일 앱에서 직접 열면 데이터 로딩이 제한될 수 있습니다.

권장 방법:

- GitHub Pages
- VS Code Live Server
- 간단한 로컬 HTTP 서버

Python이 설치되어 있다면:

```bash
python -m http.server 8000
```

그다음 브라우저에서:

```text
http://localhost:8000
```

# 11. GitHub Pages 배포

저장소 루트에 최소한 다음 파일이 있어야 합니다.

```text
saju-love-match/
├── index.html
├── idols.json
└── README.md
```

GitHub에서:

1. 저장소의 `Settings`
2. 왼쪽 메뉴 `Pages`
3. Source에서 `Deploy from a branch`
4. Branch: `main`
5. Folder: `/(root)`
6. `Save`

배포 주소 예시:

```text
https://사용자아이디.github.io/saju-love-match/
```

새 커밋을 올리면 Pages가 자동으로 다시 배포됩니다.

- 노란색: 대기 또는 배포 중
- 초록색: 배포 완료
- 빨간색: 배포 실패

# 12. 프로젝트 파일 구조

```text
saju-love-match/
├── index.html          # 화면, 스타일, 사주 계산, 궁합 점수 계산
├── idols.json          # 인물 데이터
├── README.md           # 프로젝트 설명
├── docs/
│   ├── SCORING.md      # 점수 구조 상세 설명
│   └── DATA.md         # 데이터 추가 가이드
└── tools/
    └── validate.js     # 데이터 검사 도구
```

`docs`와 `tools` 폴더는 선택 사항이며, 사이트 실행 자체에는 `index.html`과 `idols.json`이 핵심입니다.
